# Contributing to EPIC

Thank you for your interest in contributing to the **Efficient Physical Intelligence Compute** (EPIC) project! We welcome contributions of all kinds — bug reports, feature requests, documentation improvements, and code.

## How to Contribute

### Reporting Issues

- Search [existing issues](https://github.com/issues?q=org%3Aepic-proj+is%3Aopen) before opening a new one.
- Use the provided issue templates to ensure all necessary information is included.
- Be as specific as possible: include steps to reproduce, expected behavior, and actual behavior.

### Submitting Pull Requests

1. Fork the repository and create a branch from `main`.
2. Make your changes, keeping commits focused and well-described.
3. Add or update tests for your changes where applicable.
4. Ensure all existing tests pass before submitting.
5. Open a pull request using the provided PR template.
6. Address any review feedback promptly.

### Code Style

- Follow the coding conventions established in each repository.
- Keep changes minimal and focused on the problem at hand.
- Document public APIs and non-obvious logic.

## Community Standards

All contributors are expected to follow our [Code of Conduct](CODE_OF_CONDUCT.md). We are committed to maintaining a welcoming and respectful community.

## Questions

Open a [Discussion](https://github.com/orgs/epic-proj/discussions) if you have questions or want to share ideas.
